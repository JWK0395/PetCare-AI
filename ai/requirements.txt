# PetCare AI Agent 서비스 런타임 의존성
#
# ── 설치 순서 ────────────────────────────────────────────────────────────
#   cd ai
#   python -m venv .venv
#   .venv\Scripts\activate
#   pip install -r requirements.txt
#
# petcare_ai (RAG + LangGraph 본체)는 이 폴더 안(ai/petcare_ai)에 함께 있으며,
# 이 requirements 가 그 런타임 의존성까지 모두 포함한다. 서비스를 ai/ 에서 실행하면
# app/petcare_bridge.py 가 ai/ 를 sys.path 에 얹어 `petcare_ai` 를 import 한다.
# 다른 위치에서 실행할 때는 PETCARE_AI_PATH 로 상위 폴더를 지정하거나
# `pip install -e .[openai]` 로 설치한다.
#
# `[openai]` extra 는 langchain-openai + tavily-python 을 끌어온다.
# `[local]` extra(sentence-transformers · torch)는 **설치하지 않는다** — 아래 참고.
#
# ── AWS 프리티어(t2/t3.micro · RAM 1GB) 전제 ─────────────────────────────
#   torch / sentence-transformers / transformers 를 넣지 않는다.
#   torch 휠만 800MB 이상이고 bge-m3 로딩에 2GB+ 메모리가 필요해 인스턴스가 죽는다.
#   임베딩은 OpenAI API 로 처리한다 → ai/.env 에 EMBEDDING_BACKEND=openai
#   FAISS index 는 Colab 등에서 미리 만들어 배포하고 PETCARE_INDEX_DIR 로 경로만 알려준다.
#
# 이 파일에 없어도 되는 것: pypdf(진단서 PDF 텍스트 추출은 메인 서버가 한다),
# chromadb(faiss 로 통일), langchain-community(사용 안 함).
